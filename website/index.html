About the Show

Fully AI-generated, interview-focused system design discussions

System Design Podcast is an experiment in AI-generated technical education. Two AI hosts — Alex and Blake — discuss a system design topic each episode, covering real-world architectures, trade-offs, and the kind of thinking that matters in engineering interviews and on the job.

Each episode is fully generated by a Python pipeline: research phase, script writing, quality review, multi-voice audio synthesis, and architecture diagram generation — all automated. The result is a concise, interview-ready breakdown in about 10 minutes.

Topics come from real-world system design classics: URL shorteners, rate limiters, distributed caches, search engines, notification systems, and more. Real engineering blog references are woven in throughout.

Episode Format

Every episode follows five structured segments, designed to mirror how great engineers think through system design.

  • Introduction Set the stage — what is this system and why does it matter?
  • Core Architecture High-level design, key components, and how data flows.
  • Deep Dive The hard parts — scaling, consistency, performance trade-offs.
  • Real-World Examples How Google, Netflix, Uber, and others actually built it.
  • Interview Angles What interviewers look for and common pitfalls to avoid.
